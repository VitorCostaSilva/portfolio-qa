# Bug Report

Esta pasta reúne todos os **relatórios de defeitos (bug reports)** identificados durante as atividades de teste deste portfólio.

---

## Propósito

Documentar defeitos encontrados em aplicações testadas de forma estruturada, clara e rastreável — seguindo as boas práticas do mercado de QA.

Um bom bug report é uma das habilidades mais valorizadas em QA: ele precisa ser objetivo o suficiente para que o desenvolvedor consiga reproduzir e corrigir o problema sem precisar perguntar nada.

---

## Estrutura de um Bug Report

Cada bug report registrado neste portfólio segue os campos abaixo:

- **ID** — Identificador único do bug
- **Título** — Descrição curta e objetiva do problema
- **Ambiente** — Browser, sistema operacional, versão da aplicação
- **Severidade** — Impacto técnico do bug (Critical, High, Medium, Low)
- **Prioridade** — Urgência de correção sob perspectiva de negócio
- **Pré-condições** — Estado necessário antes de reproduzir
- **Passos para reprodução** — Passo a passo detalhado
- **Resultado obtido** — O que aconteceu de fato
- **Resultado esperado** — O que deveria acontecer
- **Evidências** — Prints, vídeos ou logs comprobatórios

---
